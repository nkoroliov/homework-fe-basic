Перелік основних команд:

  • git config --global user.name blabla - встановлює ім’я в локальний репозиторій
  • git config --global user.email blabal@bla - встановлює пошту в локальний репозиторій
  • git init - ініціалізує пустий репозиторій, тобто створить новий підкаталог .git, який містить всі необхідні файли вашого репозиторія — скелет Git-репозиторія.
  • git status - показує статус файлів в робочому репозитарію
  • git add - додає існуючі файли з робочого репозитарію до індексу
  • git commit - записує зміни до репозиторію із певним інформаційним повідомленням
  • git log - перевірка стану файлів
  • git checkout - використовується для переключення між гілками
  • git branch - може виводити список існуючих гілок, створювати нову гілку, вилучати гілки та перейменовувати їх
  • git push - відправляє локальні зміни на віддалений сервер
  • git pull - завантажує з віддаленого сервера на локальний репозиторій
  • git merge - використовується для зливання однієї чи більше гілок до поточної гілки
  • git clone - клонування Git репозиторія

Що таке .gitignore файл?

Файл .gitignore визначає, які файли та папки не потрібно передавати на віддалений сервер.

Advanced

  • git revert - замість видалення комміту з історії проекту ця команда скасовує внесені раніше зміни та додає новий коміт з отриманим вмістом
  • git reset - скасування попередньо введених команд
  • git rebase - може взяти всі зміни, що були в комітах одної гілки та відтворити їх на іншій гілці
  • git cherry-pick - використовується, щоб скопіювати впроваджені в одному коміті Git зміни, та застосувати їх як новий коміт на необхідній гілці
  • git stash - використовується для тимчасового збереження роботи поза комітом
